# 食譜收藏夾系統 PRD (產品需求文件)

## 1. 產品概述 (Product Overview)
這是一個專為「個人」打造的食譜收藏與管理系統。讓使用者能有系統地儲存、整理及查找日常食譜，並解決日常「不知道要吃什麼」或「烹飪時需要換算食材份量」的痛點。

## 2. 目標用戶 (Target User)
- **主要用戶**：我自己 (個人使用)
- **使用情境**：
  - 看到不錯的食譜想要記錄下來時。
  - 要買菜、準備下廚前，需要確認食材列表和份量時。
  - 用餐時間不知道要吃什麼，需要系統推薦時。

## 3. 核心功能需求 (Core Features)

### 3.1 結構化新增食譜
- **功能描述**：讓使用者能以標準化的格式輸入食譜資訊，以便後續查詢和統一閱讀格式。
- **資料欄位包含**：
  - **基本資訊**：標題 (必填)、分類 (如：主食、配菜、湯品、甜點)、料理總時間。
  - **份量資訊**：基礎份量 (例如：2 人份)。
  - **食材清單**：支援個別輸入「食材名稱」、「數量」、「單位」。
  - **步驟說明**：支援多步驟列表 (例如：Step 1, Step 2...)。
  - **標籤 (Tags)**：(選配) 允許自行定義標籤，例如「懶人料理」、「低卡」、「快速上桌」。
- **介面需求**：表單設計需直覺，支援動態增加食材與步驟輸入框。

### 3.2 快速查詢
- **功能描述**：提供高效的搜尋與篩選方式，讓使用者能迅速找到需要的食譜。
- **查詢方式**：
  - **關鍵字搜尋**：可搜尋食譜標題、食材名稱或標籤。
  - **分類/標籤篩選**：透過點擊標籤或分類快速過濾列表。

### 3.3 預覽體驗
- **功能描述**：提供專為「邊看邊煮」情境設計的乾淨閱讀介面。
- **介面需求**：
  - 排版清晰，字體適中易讀，避免反光或文字過小看不清。
  - 明確區分「食材區」與「步驟區」。
  - (選配) 步驟打勾機制，方便在料理過程中標記「已完成」的步驟，避免忘記或重複操作。

### 3.4 轉換份量
- **功能描述**：解決食譜預設份量與當下實際食用人數不符的問題。
- **互動方式**：
  - 在預覽頁面提供一個「調整份量」的控制器 (例如原為 2 人份，可透過 `-` / `+` 按鈕切換成 1 人份或 4 人份)。
  - 切換後，食材清單中的「數量」會自動依照比例即時重新計算並顯示 (例如：原需 2 顆蛋，改為 4 人份後自動顯示 4 顆蛋)。

### 3.5 隨機挑選「今天吃什麼」
- **功能描述**：幫助解決選擇困難症，隨機推薦一道食譜。
- **互動方式**：
  - 提供一個顯眼的「幫我決定」或「抽食譜」功能入口。
  - 點擊後，系統隨機從資料庫（或依據特定分類/標籤）抽出並展示一道食譜。

## 4. 非功能需求 (Non-Functional Requirements)
- **跨裝置適配 (RWD)**：高度重視「手機端」體驗（因常在廚房使用手機看食譜），必須完美支援手機版網頁版型。
- **操作流暢度**：注重前端互動的即時性，尤其是在轉換份量、篩選與隨機挑選時需有平滑的過渡。
